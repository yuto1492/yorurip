# LLM_STRATEGY.md — ヨルリプ LLM選定とコスト戦略

ヨルリプで使用するLLM APIの選定方針、フェーズ別の使い分け、コスト最適化戦略をまとめる。
2026年4月時点の各社公開料金に基づく。

## 結論サマリ

- **メインLLM: Claude (Anthropic)** に統一
- **フェーズ1 (ローンチ〜PMF確認)**: 全ユーザー Claude Opus 4.7
- **フェーズ2 (課金開始)**: 無料 = Sonnet 4.6 / 有料 = Opus 4.7
- **フェーズ3 (DAU 数千超)**: 無料 = Haiku 4.5 / プレミアム = Opus 4.7
- **将来オプション**: 量産時の無料層を Gemini 3 Flash に切替検討

Anthropic統一の理由は、SDK・プロンプト・キャッシュ運用を一本化でき、1人開発の認知負荷とメンテコストを最小化できるため。

## なぜClaude Opus 4.7をローンチ期に使うか

ヨルリプの生成品質は **「人間っぽさ」「絵文字や口語の自然さ」「指示追従の精度」** で評価される。
Arena IFスコアでOpus 4.7が1500、GPT-5.4が1470、Sonnet 4.6が1479。
コンカフェ向けは特に **「アイドル文体」「砕けた敬語」「絵文字の自然な多用」** が要求され、ニュアンス再現力で優位なClaude系がフィット。

ローンチ期は口コミ拡大が最重要で、出力品質の劣化はそのまま信頼失墜につながる。トラフィックが少ない期間はコスト絶対額も小さいため、品質を最優先する。

## なぜOpenAI/Geminiをメインに採用しないか

OpenAI: GPT-5.5は出力単価がOpus 4.7より高く、GPT-5.4でも価格性能比でSonnet 4.6に劣る。日本語の口語生成でClaudeに僅差で劣る報告が多い。

Gemini: Gemini 3.1 Pro は安いが、絵文字や砕けた表現がフォーマル寄りになりやすく、コンカフェ的な口語の自然さで不利。Gemini 3 Flash は **量産フェーズの第二候補としては極めて強力**。

## フェーズ別の運用方針

### フェーズ1: ローンチ〜PMF確認

```
全ユーザー → Claude Opus 4.7
```

- カルテ抽出のTool Useも含めて全部Opusで通す
- ユーザー数が数十〜数百のうちはコスト絶対額が小さいので最高品質を提供
- ここでの口コミ品質が、後のスケールを決める

### フェーズ2: 課金プラン開始

```
無料ユーザー  → Claude Sonnet 4.6
有料ユーザー  → Claude Opus 4.7
カルテ抽出   → Sonnet 4.6 (Tool Useはフォーマット重視)
口調特徴抽出 → Sonnet 4.6 (バックグラウンド処理)
```

### フェーズ3: DAU 数千超

```
無料ユーザー    → Claude Haiku 4.5
スタンダード    → Claude Sonnet 4.6
プレミアムプラン → Claude Opus 4.7
バックグラウンド → Haiku 4.5 + Batch API
```

### 将来オプション

無料層のコストがさらに重くなった場合、Gemini 3 Flash ($0.50/$3) への切替を検討。
日本語の口語品質を再評価してから切替する。

## プライシング再考(コンカフェ追加を反映)

コンカフェ嬢の時給帯(¥1,300〜1,600)と、学生・Wワーカー比率の高さを考慮し、当初設計より価格帯を下げる。

| プラン | 月額 | 顧客登録 | 生成回数 | ターゲット |
|---|---|---|---|---|
| 無料 | ¥0 | 10人 | 月20回 | お試し・口コミ拡散 |
| **ライト** | **¥980** | 50人 | 月150回 | コンカフェ嬢メイン層 |
| **スタンダード** | **¥2,480** | 200人 | 無制限 | キャバ・ホスト・売れっ子コンカフェ |
| **プロ** | ¥4,980 | 無制限 | 無制限 + 優先処理 | トップキャスト |

旧設計(¥1,480/¥2,980/¥5,980)より下げているが、コンカフェ嬢の所得帯では ¥980 が転換ラインと判断。
キャバ嬢・ホストは ¥2,480 のスタンダードで取りに行く。

### LLMコスト視点での持続可能性

スタンダード ¥2,480 / 月、Sonnet 4.6 で月100回利用を想定すると LLM原価は約 ¥400。粗利率約84%。
無料プラン Haiku 4.5 で月20回 = 約 ¥30 / 人。無料ユーザーを抱えるコストは十分許容範囲。

## プロンプトキャッシュ最適化 (必須)

Claude API はキャッシュヒット時の入力料金が **base inputの10% (90%オフ)**。

### キャッシュ対象として設計するもの

1. **業種ペルソナ + チャネル別文体ガイド** (システムプロンプトの大部分) → 業種x チャネルの組み合わせごとにキャッシュ。20通り(5業種 x 4チャネル)分のキャッシュエントリが理論上発生
2. **ユーザープロフィール + 口調特徴 (3層)** → 同一ユーザー内で共通。**Few-shotサンプル(層3)を含めても固定なのでキャッシュ対象**
3. **対象顧客カルテ** → 同じ顧客への連続生成でヒット

### 実装方針

- システムプロンプトは `[業種ペルソナ] + [チャネルガイド] + [安全モード]` の組み合わせで決定論的に確定し、`cache_control: { type: "ephemeral" }` を付与
- ユーザー情報ブロックも同様にcache_control付与（3層のtone_features全体を含む）
- リクエストごとに変わる部分(シーン指示、相手文面、今日の出来事)のみ末尾に配置

### Few-shotサンプル追加によるコスト影響

口調学習の3層構造化により、ユーザー情報ブロックにFew-shotサンプル3〜5本（500〜800トークン）が追加される。

- キャッシュなし時の追加コスト: Opus 4.7で **+$0.0004 / 回（約0.06円）**
- キャッシュヒット時の追加コスト: **+$0.00004 / 回（約0.006円）**

口調再現精度の大幅改善（実テストで確認済）に対してほぼ無視できるコスト。

### 効果見積

ヨルリプの典型リクエスト(入力5,500のうち、システム1,500 + ユーザー2,000 + 顧客1,500 + 可変500)で、5,000トークンがキャッシュヒット可能。
実効入力コスト: $5/M × 0.5K + $0.5/M × 5.0K = **$0.005**(キャッシュなし $0.0275 の約18%)。
Few-shotを足してもキャッシュ前提なら実効コストはほぼ変わらない。

## SDKと実装メモ

- 公式SDK: `@anthropic-ai/sdk` (Node/TypeScript)
- Nitroサーバー側で `process.env.ANTHROPIC_API_KEY` を読む。クライアント露出禁止
- ストリーミング応答 (Server-Sent Events) でユーザー体感速度を改善
- Tool Use は同じSDKで `tools` パラメータを渡す
- モデルID切替: `claude-opus-4-7` / `claude-sonnet-4-6` / `claude-haiku-4-5-20251001`
- 業種x チャネルx 安全モードの組み合わせをシステムプロンプト生成関数で動的に組み立てる

## レート制限とフォールバック

- Anthropic側のレート制限はティア制 (Tier 1〜4)。立ち上げ期はTier 1〜2想定
- 過負荷時のフォールバック順: Opus 4.7 → Sonnet 4.6 → Haiku 4.5
- 完全障害時のセカンダリ: GPT-5.4 を待機 (フォーマットだけ吸収するアダプタ層)

## 監視とコスト管理

- Sentry: API呼び出しエラー・レイテンシ監視
- Anthropic Console: 日次トークン消費量を確認
- 自前ダッシュボード: ユーザーあたり月間トークン消費を `generation_history` から集計
- 不正利用 (短時間大量生成) は `/api/generate` のレート制限 (10回/分/ユーザー) で抑止
